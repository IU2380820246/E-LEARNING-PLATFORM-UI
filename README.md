DESCRIPTION:
This project is a basic E-Learning platform front-end website created to understand how course-based websites guide users through lessons. The purpose of this project is to practice page structure, navigation flow, and small user interactions using only client-side technologies. It is not designed as a full learning management system and therefore does not include login accounts, servers, or database storage.
The website contains multiple pages such as Home, Courses, Video Player, and Dashboard. The Home page acts as the entry point and provides navigation links to different sections. The Courses page displays available lessons using simple cards so users can easily select a topic. When a lesson is opened, the Video Player page shows an embedded learning video along with its title and short description.
To simulate learning progress, a small completion tracking feature is included. When the user clicks the mark complete button, the completion percentage increases. The value is saved in browser local storage so refreshing the page does not reset the progress. This storage works only on that device and demonstrates remembering user activity without login systems.
The interface is kept simple and readable. Clear buttons, balanced spacing, and minimal colors help users focus on the content. The layout adjusts automatically on different screen sizes so the website works properly on both mobile phones and desktop computers.
While developing this project, I practiced connecting multiple pages, updating elements using JavaScript, and storing small data inside the browser. The project helped in understanding how educational interfaces move users step by step through lessons using basic functionality.
Overall, this project represents a beginner-level learning website that demonstrates course viewing and progress indication using only front-end technologies.

TECHNOLOGY
HTML: page structure and navigation
CSS: layout styling and responsiveness
JavaScript: progress tracking using local storage
